## English

Thank you for your interest in contributing to this project! This documentation is
published under the CC BY-SA 4.0 licence, which means all contributions are welcome
as long as they respect the same spirit of openness and attribution.

### What you can contribute

- **Technical corrections** — errors in thermodynamic principles, COP values, dates,
  or technical descriptions.
- **Historical additions** — sources, references, or context that enrich the narrative.
- **Translations** — improving or extending the French/English versions, or adding a
  new language under `docs/`.
- **Diagrams and illustrations** — redrawn or improved SVG/PNG diagrams (open formats
  preferred).
- **Typos and wording** — any improvement to clarity, readability, or style.

### How to contribute

1. **Fork** this repository on GitHub.
2. Create a **branch** with a descriptive name:
   `git checkout -b fix/cop-values-1980s` or `git checkout -b add/fr-section-compressors`
3. Make your changes and **commit** with a clear message in English or French.
4. Open a **Pull Request** against the `main` branch, describing what you changed and why.

### Style guidelines

- Keep the bilingual structure: if you edit the French version, please update the English
  version too (or flag it in the PR so someone else can).
- Diagrams should be provided in **SVG** or high-resolution **PNG** (≥ 150 dpi).
- References to published sources should follow the format:
  `Author, *Title*, Publication, Year — [link if available]`
- Avoid reproducing copyrighted text verbatim; paraphrase and cite instead.

### Code of conduct

This is a technical and historical documentation project. Please keep discussions
respectful and on-topic. Constructive criticism is welcome; personal attacks are not.

### Reporting issues

Found an error or have a suggestion? Open an **Issue** on GitHub with:
- A clear title
- A description of the problem or suggestion
- The relevant section or file

## Français

Merci de l'intérêt que vous portez à ce projet ! Cette documentation est publiée sous
licence CC BY-SA 4.0, ce qui signifie que toutes les contributions sont les bienvenues,
dans le respect du même esprit d'ouverture et d'attribution.

### Ce à quoi vous pouvez contribuer

- **Corrections techniques** — erreurs sur les principes thermodynamiques, valeurs de COP,
  dates ou descriptions techniques.
- **Ajouts historiques** — sources, références ou éléments de contexte qui enrichissent
  le récit.
- **Traductions** — améliorer ou compléter les versions française/anglaise, ou ajouter une
  nouvelle langue sous `docs/`.
- **Schémas et illustrations** — schémas SVG/PNG redessinés ou améliorés (formats ouverts
  préférés).
- **Coquilles et formulations** — toute amélioration de la clarté, de la lisibilité ou du style.

### Comment contribuer

1. **Forkez** ce dépôt sur GitHub.
2. Créez une **branche** avec un nom descriptif :
   `git checkout -b fix/valeurs-cop-1980` ou `git checkout -b ajout/section-compresseurs`
3. Effectuez vos modifications et **committez** avec un message clair en français ou en anglais.
4. Ouvrez une **Pull Request** vers la branche `main`, en décrivant ce que vous avez modifié
   et pourquoi.

### Conventions de style

- Respectez la structure bilingue : si vous modifiez la version française, mettez également
  à jour la version anglaise (ou signalez-le dans la PR pour qu'une autre personne s'en charge).
- Les schémas doivent être fournis en **SVG** ou en **PNG** haute résolution (≥ 150 dpi).
- Les références à des sources publiées suivent le format :
  `Auteur, *Titre*, Publication, Année — [lien si disponible]`
- Évitez de reproduire verbatim des textes protégés par le droit d'auteur ; paraphrasez et citez.

### Code de conduite

Il s'agit d'un projet de documentation technique et historique. Merci de maintenir des
échanges respectueux et centrés sur le sujet. La critique constructive est la bienvenue ;
les attaques personnelles ne le sont pas.

### Signaler un problème

Vous avez trouvé une erreur ou avez une suggestion ? Ouvrez une **Issue** sur GitHub en
précisant :
- Un titre clair
- Une description du problème ou de la suggestion
- La section ou le fichier concerné
